# LSP Semantic Diagnostics on Every Write

Run a real language server against edited files and surface errors before the next turn.

## What Done Looks Like

- [ ] Install `pyright`, `typescript-language-server`, `vscode-json-languageserver`
- [ ] After every write/edit, spawn relevant LSP server in single-file mode
- [ ] Capture diagnostics, filter to NEW errors only
- [ ] Surface as `[[diagnostic]]` block in tool result
- [ ] If error count > threshold, pause and ask user before continuing

## Why This Matters

We ship broken code downstream. Type errors, undefined symbols, missing imports — all caught immediately. Goes WAY beyond basic syntax lint.

- New skill: `lsp-diagnostic/` or integrate into write/edit skills
