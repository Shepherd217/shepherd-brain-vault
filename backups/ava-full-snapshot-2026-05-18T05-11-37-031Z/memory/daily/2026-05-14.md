### Key Findings

**HermesAgent Dashboard (v0.13.0):**
- 6 surfaces: CLI, TUI (React Ink + JSON-RPC over stdio), Gateway (20+ platforms), ACP (IDE integration), Batch, API Server, Web Dashboard
- Web Dashboard pages: Overview/Health, Plugins, Profiles, Analytics, Settings, One-click update, i18n (7 locales)
- **Crown jewel:** Multi-Agent Kanban Board — "one install, many kanbans", durable tasks, multiple workers, heartbeat system, reclaim mechanism, zombie detection, per-task retry budgets, hallucination recovery, task handoff
- Profile system: Complete isolation per profile (own config, memory, sessions, skills, gateway)
- SQLite session storage with FTS5, lineage tracking, per-platform isolation
- Cron system: `hermes cron` with `no_agent` watchdog mode

**OpenCode Parallel Agents:**
- **Core isolation primitive: Git worktrees** — industry standard by 2026 (Claude Code, Cursor, Hermes, OpenCode all support)
- CLI flag `--worktree` / `-w` auto-creates isolated branch+workspace at `.opencode/worktrees/`
- Subagent worktree isolation via `isolation: worktree` directive in subagent YAML
- `/batch` command auto-creates 5-30 worktrees for large parallel changes
- File coordination layer for concurrent sibling subagents to prevent edit clobbering
- `--teammate-mode` options: auto, in-process, tmux
- Known bugs: Global team storage race condition, submodule worktree bug

**Worktree comparison table (OpenCode):**
- Git worktree: ~1s creation, working tree only overhead, no port/DB isolation, standard git merge
- Full clone: ~30-120s, full repo copy, no isolation, manual sync
- Docker: ~10-60s, image layers, full isolation, volume mount complexity
- Hybrid emerging standard: worktree per agent + lightweight container per worktree
