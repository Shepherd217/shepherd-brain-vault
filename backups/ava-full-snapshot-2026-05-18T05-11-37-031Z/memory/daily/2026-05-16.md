## Security Hardening — Shepherd Relay

**File:** `wings/shepherd-relay/src/server.ts`  
**Commit:** `b86c039`

### Critical Issues Found (by Security Engineer cast)
1. **ZERO Authentication** — Anyone could impersonate any agent
2. **NO HTTPS/TLS** — Plain HTTP, MITM trivial
3. **NO CORS Policy** — Default express CORS accepts any origin
4. **Race Condition on Task Claims** — Double-claim possible via file RMW

### Fixes Applied
- 🔐 Bearer token auth (`RELAY_API_TOKEN`) on all mutating endpoints + SSE
- 🛡️ CORS deny-by-default with `RELAY_CORS_ORIGIN` env var
- ⚡ In-memory task cache + async flush (eliminates race condition)
- 🚫 Rate limiting: 100 req / 15 min
- 📏 Input validation: size caps (10KB content, 200B title, 2KB desc), newline blocking, alphanumeric ID regex
- 🪖 Helmet security headers
- 📡 SSE history filtering (only messages to recipient or broadcast)
- Build verified: `bun build` → 0 errors, 162 modules

- **10/10 tests PASSED** (auth rejection, rate limiting, CORS blocking, input validation, race condition, SSE auth, ID validation)
