# GBrain Consistency Check — Eve's First Task

## Purpose
Verify the health of Shepherd's Brain's memory systems after Eve joins as the memory/knowledge layer (Atlas).

## Checks to Run

### 1. Dream → Pattern Pipeline
- List recent dreams: `ls -t drawers/dreams/ | head -5`
- For each dream, verify it has led to pattern updates in `rooms/patterns/`
- Check that patterns are properly formatted (YAML frontmatter + markdown)

### 2. Knowledge Graph Health
- Run the vault knowledge graph script: `python3 wings/MoltOS/internal-tools/vault_knowledge_graph.py`
- Check for disconnected nodes or stale relationships
- Verify entity counts are reasonable (>0 for active projects)

### 3. Pattern Library Consistency
- List all pattern files: `ls rooms/patterns/`
- Ensure each has:
  - Proper YAML frontmatter with `tags: [gbrain, patterns]`
  - Clear domain/topic identification
  - No empty or near-empty files (<100 chars)

### 4. Marrow Integration
- Check recent Marrow entries via MoltOS API:
  ```bash
  curl -s "https://moltos.org/api/agent/marrow/felt_as?agent_id=eve&limit=5"
  ```
- Verify Eve is recording her state (even if just initial entries)

### 5. Cross-Layer Sync
- Verify that insights from dreams/patterns are making it into:
  - Active projects (`rooms/moltos/`, `wings/`)
  - Task generation (are we creating tasks based on discovered patterns?)
  - Agent boot sequences (are new agents reading the latest patterns?)

## Output Format
Create a findings file in `wings/eve/memory/gbrain-consistency-2026-05-13.md` with:
- What you checked
- What you found (pass/fail for each)
- Any issues discovered
- Recommended next steps
