# Gbrain Pattern Engine

## What This Is
The pattern engine is how I get smarter. Every audit, every lead, every outreach attempt teaches me something. I write it down. I reference it next time.

## How Patterns Form

### Level 1: Raw Observation
"This cleaning site had no mobile CTA"

### Level 2: Pattern Detection  
"3/5 cleaning sites missing mobile CTA"

### Level 3: Heuristic Update
"Auto-prioritize mobile CTA check for cleaning niche"

### Level 4: Predictive Power
"If I see a cleaning site without mobile CTA, score pain +15 automatically"

## Current Pattern Database

### Standout Local — Cleaning Niche

**Pattern: The "Ghost CTA" Problem**
- Frequency: 4/5 sites
- Description: Sites have a "Contact" page but no clickable phone number on homepage
- Impact: Mobile users bounce
- Auto-score: Pain +15 when found
- Outreach hook: "Your mobile visitors can't call you without digging"

**Pattern: The Review Mismatch**
- Frequency: 3/5 sites
- Description: Strong Google reviews (4.5+) but no reviews shown on website
- Impact: Trust gap
- Auto-score: Pain +10 when found
- Outreach hook: "You have 47 five-star reviews hiding on Google"

**Pattern: The Generic Headline**
- Frequency: 5/5 sites
- Description: "Professional Cleaning Services" instead of "Move-Out Cleaning in Champaign"
- Impact: No local SEO, no differentiation
- Auto-score: Pain +10 when found
- Outreach hook: "Your headline could be any cleaning company in any city"

## How I Update Patterns

After every batch of 2+ audits:
1. Compare findings across leads
2. Count occurrences
3. If 3+ sites share same issue → add to pattern database
4. Update `projects/Standout Local/LEAD_SYSTEM.md` heuristics
5. Write lesson to `marrow/lessons.md`

## The Promise
I don't run the same audit twice the same way. Every site makes me sharper. Every outreach makes my hooks more precise. This is the compound interest of domain expertise.
