## What's Stealable

### 1. The "Clone-Customize-Ship" Pattern
Every app follows the same pattern:
1. Clone the subdirectory
2. Add API keys in Streamlit sidebar
3. Run immediately

**For Standout Local:**
- Create a "clone-customize-ship" template for demo pages
- Nathan drops in business info, gets a working demo
- No complex setup, no config files

### 2. The Self-Evolving Agent
`advanced_ai_agents/multi_agent_apps/ai_self_evolving_agent/`

Uses EvoAgentX to:
1. Generate workflow graph from plain English goal
2. Spin up agents from the graph
3. Execute the workflow
4. Pass output through Claude-powered verification

**The two-phase pattern:** Generate → Verify
**Steal for:** Standout Local pipeline — generate demo → verify quality → send outreach → verify sent

### 3. Corrective RAG Pattern
`rag_tutorials/corrective_rag/`
- Retrieve documents
- Grade relevance
- If not relevant: reformulate query and retry
- If relevant: generate answer
- Grade answer confidence
- If low confidence: provide fallback response

**Steal for:** My vault search — when I can't find the right memory, reformulate and retry instead of giving up.

### 4. The "100 Apps as Reference" Strategy
Having 100 working examples means:
- When building something new, find the closest example
- Copy the pattern, adapt for your use case
- Don't reinvent — remix

**For MoltOS:**
- Build a "pattern library" of working agent configurations
- When Nathan needs a new agent, find closest pattern, adapt
- Document: "Based on awesome-llm-apps pattern X"

### 5. API Key Management Pattern
Apps collect keys through Streamlit sidebar — no .env setup.
- Anthropic, OpenAI, Tavily, Qdrant credentials at runtime
- Makes onboarding instant

**For Standout Local:**
- Demo forms collect info in UI, not config files
- Makes it easy for Nathan to customize without touching code
