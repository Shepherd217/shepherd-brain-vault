# 🎭 Roster — Agent Command Center

The Roster is your **mission control** for the agency-agents library. It shows who's available, what they've done, and how to deploy them.

## 🎯 What the Roster Does

- **Browse** — List all 70+ agents by division, skill, or vibe
- **Search** — Find the perfect agent for any task
- **Cast** — Deploy an agent with one command
- **Track** — See cast history, success rates, and output quality
- **Stats** — Usage analytics per role, per day, per task type

## 📂 Data Sources

The Roster reads from:

1. `wings/agency-agents/` — The raw agent profile files
2. `drawers/entries/YYYY-MM-DD-cast-log.md` — Cast execution logs
3. `drawers/entries/YYYY-MM-DD-openshell-log.md` — Sandboxed cast logs
4. (Optional) `drawers/roster/` — Generated roster files

## ⚡ Commands

### `roster list` — Browse Available Agents

```bash
roster list                          # All agents
roster list --division=engineering   # Only Engineering
roster list --division=marketing     # Only Marketing
roster list --vibe=strategic         # Strategic thinkers
roster list --emoji=🔒               # Security-focused
```

### `roster find` — Search by Keyword

```bash
roster find "security"              # Any agent mentioning security
roster find "api"                   # API-related agents
roster find "wechat"                # WeChat specialists
roster find "frontend"              # Frontend developers
```

Returns ranked matches with relevance score.

### `roster cast` — Deploy an Agent

```bash
roster cast security-engineer "Audit wings/shepherd-relay/ for auth flaws"
```

This is a shorthand that:
1. Finds the profile in `wings/agency-agents/`
2. Calls `sessions_spawn` with the profile as system prompt
3. Logs the cast to today's log file
4. Returns the deliverable

### `roster log` — View Cast History

```bash
roster log --today                   # Today's casts
roster log --role=security-engineer  # All casts by role
roster log --status=success          # Only successful casts
roster log --last=7                  # Last 7 days
```

### `roster stats` — Usage Analytics

```bash
roster stats                         # Overall stats
roster stats --role=code-reviewer    # Per-role stats
roster stats --division=engineering  # Per-division stats
roster stats --period=week           # Weekly breakdown
```

## 🏗️ Implementation Details

### Generating the Roster Index

The roster index is a JSON file auto-generated by scanning `wings/agency-agents/`:

```json
{
  "generatedAt": "2026-05-16T12:00:00Z",
  "totalAgents": 70,
  "divisions": {
    "engineering": {
      "count": 25,
      "agents": [
        {
          "id": "engineering-security-engineer",
          "name": "Security Engineer",
          "emoji": "🔒",
          "vibe": "protective",
          "description": "Find and fix vulnerabilities...",
          "file": "engineering/engineering-security-engineer.md",
          "skills": ["audit", "penetration-testing", "threat-modeling"]
        }
      ]
    }
  }
}
```

Generate it with:
```bash
# Scan all .md files and extract YAML frontmatter
node scripts/generate-roster-index.js
```

### The Dashboard (HTML)

For a visual roster, generate a static HTML dashboard:

```bash
roster dashboard --output=drawers/roster/dashboard.html
```

This produces a searchable, filterable table of all agents with:
- Profile cards (emoji, color, vibe)
- Skill tags
- "Cast Now" buttons
- Usage stats (if available)

## 📝 Log Aggregation

The roster aggregates logs from multiple files:

```javascript
// Pseudocode for log aggregation
const logFiles = glob('drawers/entries/*-cast-log.md');
const logs = logFiles.flatMap(file => parseCastLog(file));

// Group by role, date, status
const stats = {
  totalCasts: logs.length,
  byRole: groupBy(logs, 'role'),
  byDate: groupBy(logs, 'date'),
  byStatus: groupBy(logs, 'status'),
  avgDuration: mean(logs.map(l => l.durationSeconds))
};
```
